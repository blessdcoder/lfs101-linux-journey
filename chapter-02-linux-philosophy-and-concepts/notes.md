## What This Chapter Is About

This chapter covers where Linux came from, what it stands for, and 
the key terms and ideas you need to know before you start using it.

---

## Linux History

Linux was created by a guy named Linus Torvalds in 1991. He was a 
student in Finland at the time and started building it as a personal 
project. He made it free and open for anyone to use, change, and share.

Before Linux, there was Unix — but it was expensive and owned by 
companies. Linux gave people a free alternative.

---

## Linux Philosophy

Linux is built around a few core ideas:

- Everything in Linux is treated as a file. Even hardware devices 
  are represented as files on the system.
- Programs are kept small and focused. Each tool does one job well. 
  You can combine them to do more complex things.
- Configuration is stored in plain text files. This makes it easy 
  to read and edit settings without special software.

---

## Linux Community

Linux is built and maintained by a global community of developers. 
Anyone can contribute. Companies like Google, Intel, and Red Hat 
also have people working on it full time.

There are forums and communities online where people help each other 
with Linux — like Stack Overflow, Ask Ubuntu, and Reddit.

## Key Linux Terms

These are words that come up a lot when working with Linux:

| Term | What It Means |
|------|--------------|
| Kernel | The core of Linux. It manages hardware and system resources |
| Shell | The interface you use to type commands and talk to the kernel |
| Distribution | A full Linux OS package built around the kernel |
| Desktop Environment | The graphical interface like GNOME or KDE |
| CLI | Command Line Interface — text based interaction |
| GUI | Graphical User Interface — visual interaction with windows |
| Repository | A server where software packages are stored and downloaded from |
| Package Manager | A tool used to install and remove software |

---

## Linux Distributions

A distribution (or distro) is a complete Linux operating system. They 
all use the same Linux kernel but differ in what software comes 
included and how things are managed.

The three main families covered in this course are:

| Family | Examples | Package Manager |
|--------|----------|----------------|
| Debian | Ubuntu, Linux Mint | apt |
| Red Hat | Fedora, RHEL | dnf or yum |
| SUSE | openSUSE | zypper |

I am using Ubuntu which is part of the Debian family.

## What I Found Interesting

The idea that everything is a file in Linux was new to me. I always 
thought files were just documents and folders. Knowing that even 
hardware is treated as a file changes how I think about the system.

---

## What Confused Me

The difference between a kernel and a shell took me a moment.
After reading it again I understand it like this:
- The kernel is the brain — it talks directly to the hardware
- The shell is the mouth — it takes what I type and passes it 
  to the kernel
